# 根本原因分析（RCA）レポート生成スキル

## トリガー
以下のキーワードで起動:
- 「根本原因分析をして」「なぜなぜ分析」「RCAレポート」
- 「不良原因を分析して」「5Why分析」「CAPA」
- 「なぜこの不良が発生したのか分析」

## 入力
品質管理アプリの「データ出力」タブ→「分析用JSONコピー」で出力したJSONデータ。
特定の不良種別や製品に絞った分析を行う場合はユーザーに確認。

## 実行手順

### Step 1: データ読込と問題特定
1. JSONデータを読み込む
2. パレート分析を実施し、最も影響の大きい不良を特定
3. 分析対象の不良種別をユーザーに確認

### Step 2: データ分析
対象不良のレコードを集計:
- 発生傾向（時系列）
- 工程別分布
- 重大度分布
- 既知の原因分布（4M分類）
- 作業者別傾向
- 検出方法別分布

### Step 3: なぜなぜ分析（5 Why）
データから得られた情報を基に、なぜなぜ分析のフレームワークを構築。
ユーザーと対話的に進める場合と、データから自動推論する場合がある。

**テンプレート**:
```
問題: [不良種別名] が [工程名] で [期間] に [件数] 件発生

なぜ1: なぜ [不良] が発生したか？
  → [直接原因]（データ根拠: XX%が[原因]に起因）

なぜ2: なぜ [直接原因] が起きたか？
  → [中間原因]

なぜ3: なぜ [中間原因] が起きたか？
  → [更に深い原因]

なぜ4: なぜ [更に深い原因] があったか？
  → [管理的原因]

なぜ5: なぜ [管理的原因] が放置されていたか？
  → [根本原因]（システム的問題）
```

### Step 4: CAPA（是正処置・予防処置）策定

**是正処置（CA: Corrective Action）**:
発生した不良に対する直接的な対策

**予防処置（PA: Preventive Action）**:
再発を防止するための仕組み的な対策

各対策には以下を含める:
- 対策内容
- 担当者（役割）
- 期限（目安）
- 効果確認方法
- 優先度（高/中/低）

### Step 5: レポート出力

以下の形式でレポートを生成:

## 出力先

**Notionページ（推奨）**:
Notion MCPを使い、上記テンプレートに沿ったページを自動生成する。

**テキスト出力（フォールバック）**:
Markdownテキストとしてユーザーに提示する。

## データ根拠に基づく推論ルール

### 原因推論のガイドライン
- 「人」が最多 → 教育・訓練体制、作業環境を重点確認
- 「機械」が最多 → 保全体制、設備年数を重点確認
- 「材料」が最多 → 受入検査体制、サプライヤ管理を重点確認
- 「方法」が最多 → 標準化レベル、手順書整備状況を重点確認
- 特定作業者に偏り → スキルマトリクス、OJT体制を確認
- 特定工程に偏り → 工程能力（Cp, Cpk）の確認を推奨
- 特定時期に集中 → 季節要因、設備変更、人員変更を確認

### 注意事項
- データが少ない場合（30件未満）は統計的な確度が低いことを明記
- 原因が記録されていないレコードが多い場合、原因記録の運用改善を提案
- 推論と事実を明確に区別して記述する
